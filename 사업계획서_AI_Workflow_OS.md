# AI Workflow OS 사업계획서

## 1. 사업 개요

### 서비스명

가칭: 의뢰노트 / 납품노트 / Flowrit

초기 검증 단계에서는 국내 소규모 사업자가 직관적으로 이해할 수 있는 `의뢰노트`를 우선 사용한다.

### 한 문장 소개

파일 기반 업무를 하는 1~5인 사업자가 고객 의뢰, 진행상황, 수정 요청, 파일 납품 링크를 하나의 흐름으로 관리할 수 있게 해주는 AI 기반 업무 운영 SaaS.

### 핵심 포지션

이 서비스는 단순 CRM, 파일 저장소, 프로젝트 관리툴, AI 챗봇이 아니다.

고객 응대, 작업 진행, 수정 요청, 파일 전달, 납품 상태를 하나의 프로젝트 흐름으로 연결하는 소규모 사업자용 Workflow OS다.

## 2. 문제 정의

1~5인 규모의 파일 기반 작업 사업자는 대부분 카카오톡, 이메일, 엑셀, 메모장, 구글 드라이브, 노션 등을 조합해 업무를 관리한다.

고객 수와 프로젝트 수가 늘어나면 다음 문제가 반복된다.

- 고객별 진행상황이 흩어진다.
- 수정 요청이 카카오톡 대화 속에 묻힌다.
- 파일 전달 링크와 버전 관리가 분산된다.
- 고객이 진행상황을 반복해서 문의한다.
- 납품 완료 여부와 잔여 작업을 한눈에 보기 어렵다.
- 담당자가 바뀌면 인수인계가 어렵다.
- 사업자가 직접 기억에 의존해 업무를 운영한다.

이 문제는 단순한 기록 문제가 아니라 고객 응대 시간 증가, 재작업, 납품 지연, 고객 불만으로 이어지는 운영 문제다.

## 3. 목표 고객

### 초기 타겟

초기 타겟은 웨딩/사진 작가 및 소규모 사진 스튜디오로 설정한다.

선정 이유:

- 고객별 프로젝트가 명확하다.
- 원본, 보정본, 최종 납품 등 파일 전달이 많다.
- 수정 요청이 반복적으로 발생한다.
- 고객의 진행상황 문의가 잦다.
- 개인 또는 소규모 팀으로 운영되는 경우가 많다.
- 월 구독형 도구보다 업무 세팅형 서비스에 반응할 가능성이 높다.

### 확장 타겟

초기 검증 후 다음 업종으로 확장한다.

- 영상 편집자
- 디자인 스튜디오
- 마케팅 대행사
- 인테리어/설계 사무소
- 웨딩 관련 업체
- 프리랜서 팀

## 4. 고객 페르소나

### 1차 페르소나: 웨딩/사진 작가

- 1인 또는 2~3인 팀
- 월 10~50건의 촬영/보정/납품 프로젝트 운영
- 카카오톡, 구글 드라이브, 엑셀, 메모앱을 혼합 사용
- 고객 응대와 실제 작업을 동시에 처리
- 수정 요청 누락과 파일 링크 재전송에 피로감을 느낌

### 주요 니즈

- 고객별 진행상황을 한눈에 보고 싶다.
- 수정 요청을 누락하지 않고 관리하고 싶다.
- 파일 납품 링크를 쉽게 정리하고 다시 보낼 수 있어야 한다.
- 고객이 진행상황을 직접 확인하게 만들고 싶다.
- 복잡한 협업툴이 아니라 바로 쓸 수 있는 업무 장부가 필요하다.

## 5. 솔루션

### 핵심 가치

고객이 “지금 어디까지 됐나요?”라고 묻기 전에 진행상황 링크로 확인하게 해준다.

### 주요 기능

1. 고객 관리
   - 고객 정보 등록
   - 고객별 프로젝트 관리
   - 연락처, 문의 내용, 계약 상태 기록

2. 프로젝트 관리
   - 프로젝트 생성
   - 진행 단계 설정
   - 마감일 관리
   - 현재 상태 표시

3. 워크플로우 템플릿
   - 업종별 기본 단계 제공
   - 사진/웨딩 예시: 문의 -> 예약 확정 -> 원본 수령 -> 1차 보정 -> 수정 요청 -> 최종 전달 -> 완료
   - 단계별 상태 변경

4. 수정 요청 관리
   - 고객 요청 등록
   - 대상 파일 또는 항목 연결
   - 우선순위, 상태, 담당자 관리
   - 완료 여부 체크

5. 파일 링크 관리
   - Google Drive, Dropbox, NAS, S3, URL 등 외부 스토리지 링크 연결
   - 파일 버전 관리
   - 납품 링크 상태 관리
   - 만료일 기록

6. 고객용 진행상황 페이지
   - 고객에게 공유 가능한 프로젝트 링크
   - 현재 단계, 예정일, 납품 링크, 수정 요청 상태 표시
   - 고객 문의를 줄이는 핵심 차별 기능

7. 메시지 템플릿
   - 카카오톡/문자/이메일로 보낼 진행상황 안내문 복사
   - 납품 안내문, 수정 요청 접수 안내문, 완료 안내문 생성

8. AI 운영 비서
   - 고객 메시지 요약
   - 수정 요청 자동 구조화
   - 오늘 할 일 요약
   - 지연 위험 프로젝트 감지
   - 고객 답변 초안 생성

## 6. 차별화 전략

### 기존 대체재

- 카카오톡
- 이메일
- 엑셀
- 구글 드라이브
- 노션
- Trello, Asana, Monday 등 범용 프로젝트 관리툴
- 일반 CRM

### 차별점

1. 파일 저장이 아니라 납품 흐름을 관리한다.
2. 범용 프로젝트 관리가 아니라 파일 기반 의뢰 업무에 특화한다.
3. 내부 관리 화면뿐 아니라 고객용 진행상황 페이지를 제공한다.
4. 카카오톡을 대체하지 않고 카카오톡 옆에서 업무 장부 역할을 한다.
5. AI를 챗봇이 아니라 운영 보조 기능으로 사용한다.
6. 소규모 사업자가 바로 쓸 수 있는 업종별 템플릿을 제공한다.

## 9. 비용 구조

### 핵심 원칙

대용량 파일을 직접 무제한 저장하지 않는다.

### 비용 최소화 전략

- 서비스는 메타데이터 중심으로 운영한다.
- 파일은 고객의 Google Drive, Dropbox, NAS, S3 등에 보관한다.
- 자체 저장소는 썸네일, 소량 첨부, 임시 파일로 제한한다.
- AI 호출은 이벤트 기반으로 최소화한다.
- 초기에는 저비용 클라우드 조합을 사용한다.

### 예상 인프라

- Frontend: Next.js
- Backend: Next.js API 또는 Node.js
- Database: PostgreSQL
- Storage: S3 또는 Cloudflare R2
- Queue/Cache: Redis 선택
- Deployment: Vercel, Railway, Render, Supabase, Neon 등

## 11. 실행 계획

### 1단계: 경험 기반 정리

기간: 1주

실행 항목:

- 과거 의뢰 업무 흐름 정리
- 반복 문제 3개 선정
- 첫 타겟 업종 확정
- 초기 서비스명 확정
- 노션 사업 대시보드 생성

성과 목표:

- 핵심 문제 1개 확정
- 유료 파일럿 제안 문구 완성

### 2단계: 노션 기반 수동 MVP

기간: 1~2주

실행 항목:

- 고객 DB 생성
- 프로젝트 DB 생성
- 수정 요청 DB 생성
- 파일 링크 DB 생성
- 고객용 진행상황 페이지 템플릿 제작
- 메시지 템플릿 제작

성과 목표:

- 실제 고객에게 보여줄 수 있는 수동 MVP 완성

### 3단계: 유료 파일럿

기간: 2~4주

실행 항목:

- 기존 네트워크 10명에게 제안
- 파일럿 고객 3명 확보
- 최소 1명 유료 결제 유도
- 실제 프로젝트 운영
- 사용 로그와 불편사항 기록

성과 목표:

- 유료 고객 1~3명
- 실제 프로젝트 5건 이상 운영
- 반복 사용 기능 확인

### 4단계: 웹 MVP 개발

기간: 4~8주

실행 항목:

- 핵심 데이터 모델 설계

성과 목표:

## 14. 주요 리스크와 대응

### 리스크 1: 고객이 돈을 내지 않을 수 있음

대응:

- 무료 사용보다 유료 파일럿을 먼저 제안한다.
- 세팅비를 통해 실제 지불 의향을 검증한다.
- 월 구독보다 업무 정리 효과를 강조한다.

### 리스크 2: 입력이 귀찮아서 사용하지 않을 수 있음

대응:

- 초기 입력 항목을 최소화한다.
- 메시지 복사, 템플릿, 상태 변경 등 반복 작업을 줄인다.
- 카카오톡 내용을 붙여넣으면 수정 요청으로 구조화하는 기능을 우선 개발한다.

### 리스크 3: 범용툴과 비교될 수 있음

대응:

- 프로젝트 관리툴이 아니라 납품 흐름 관리 도구로 포지셔닝한다.
- 업종별 템플릿과 고객용 진행상황 페이지를 차별점으로 삼는다.

### 리스크 4: 파일 저장 비용이 커질 수 있음

대응:

- 파일 직접 저장을 기본 기능으로 제공하지 않는다.
- 외부 스토리지 링크 중심으로 설계한다.
- 자체 저장은 썸네일과 임시 파일로 제한한다.

### 리스크 5: 너무 많은 업종을 동시에 타겟팅할 수 있음

대응:

- 초기에는 웨딩/사진 작가에 집중한다.
- 동일한 문제가 반복 확인된 업종으로만 확장한다.

## 15. 12개월 목표

### 0~3개월

- 노션 기반 수동 MVP 운영
- 파일럿 고객 3~5곳 확보
- 유료 결제 1~3곳 확보
- 웹 MVP 범위 확정

### 3~6개월

- 웹 MVP 출시
- 유료 고객 10~20곳 확보
- 업종별 템플릿 1~2개 완성
- 고객 사례 3개 확보

### 6~12개월

- 유료 고객 20~30곳 확보
- MRR 100만~300만원 달성
- AI 기능 일부 도입
- 정부지원사업 신청 및 선정 추진
- 정식 가격제와 온보딩 체계 확립

## 16. 결론

AI Workflow OS는 처음부터 대형 SaaS로 접근하면 실패 가능성이 높다. 반대로 특정 업종의 반복적인 운영 문제, 특히 수정 요청 누락과 납품 상태 문의를 줄이는 도구로 시작하면 빠르게 검증할 수 있다.

초기 성공의 핵심은 제품 개발보다 실제 고객의 업무 흐름에 들어가는 것이다. 먼저 노션 기반 수동 MVP와 유료 파일럿으로 돈을 내는 문제를 확인하고, 반복적으로 사용되는 기능만 웹 제품으로 개발해야 한다.

최종 목표는 1~5인 사업자가 고객 응대, 작업 진행, 수정 요청, 파일 납품을 팀처럼 운영할 수 있게 만드는 AI 기반 업무 운영 플랫폼이 되는 것이다.
