## 배경 및 목적

1~5인 규모의 파일 기반 사업자(웨딩·사진 작가 등)는 카카오톡, 이메일, 구글 드라이브, 엑셀을 조합해 고객 의뢰, 수정 요청, 납품 파일을 관리하면서 반복적인 운영 문제를 겪는다.

- 수정 요청이 카카오톡 대화 속에 묻혀 누락된다.
- 납품 파일 링크와 버전이 흩어진다.
- 고객이 "지금 어디까지 됐나요?"를 반복해서 문의한다.
- 담당자가 바뀌면 인수인계가 어렵다.

Flowrit 1차 웹 MVP는 이 문제를 해결하는 첫 번째 제품이다. 고객 의뢰 접수, 프로젝트 진행 상태 관리, 수정 요청 관리, 파일·링크 관리, 고객용 진행상황 공유를 하나의 흐름으로 연결한다.

기존 `gohoc`(Next.js 고객 포털)과 `ourwedding`(운영 관리 시스템)에서 실제로 검증된 업무 흐름을 다중 워크스페이스 구조의 SaaS로 일반화한다.

## 사용자 스토리

### 사업자 (Service Provider)

- **US-001**: 사업자로서, 이메일·비밀번호로 계정을 만들고 팀 워크스페이스를 갖고 싶다.
- **US-002**: 사업자로서, 고객 정보를 등록하고 해당 고객의 모든 프로젝트를 한눈에 보고 싶다.
- **US-003**: 사업자로서, 새 프로젝트를 생성하고 업종에 맞는 워크플로우 단계를 설정하고 싶다.
- **US-004**: 사업자로서, 프로젝트의 현재 단계를 변경하고 변경 이력을 시간순으로 볼 수 있기를 원한다.
- **US-005**: 사업자로서, 고객이 제출한 의뢰와 수정 요청을 누락 없이 목록으로 관리하고 싶다.
- **US-006**: 사업자로서, 납품 파일 링크(Google Drive, Dropbox 등)를 프로젝트에 등록하고 버전을 관리하고 싶다.
- **US-007**: 사업자로서, 고객에게 공유할 진행상황 링크를 생성하여 "어디까지 됐나요?" 문의를 줄이고 싶다.
- **US-008**: 사업자로서, 카카오톡·문자로 보낼 안내 메시지를 버튼 클릭 한 번으로 복사하고 싶다.
- **US-009**: 사업자로서, 오늘 처리해야 할 작업(마감 임박, 미완료 수정 요청)을 대시보드에서 바로 확인하고 싶다.
- **US-010**: 사업자로서, 팀원을 워크스페이스에 초대하고 프로젝트 담당자로 배정하고 싶다.

### 고객 (Customer, 비로그인)

- **US-011**: 고객으로서, 사업자가 공유한 링크를 통해 내 프로젝트의 현재 단계, 예정일, 납품 파일 링크를 확인하고 싶다.
- **US-012**: 고객으로서, 의뢰 접수 시 원본 파일·참고자료·요청사항을 구조화해서 제출하고 싶다.
- **US-013**: 고객으로서, 결과물을 받은 후 수정 요청을 구조화해서 접수하고 싶다.

## 기능 요구사항

### 인증·워크스페이스

- **FR-001**: 사업자는 이메일·비밀번호로 회원가입 및 로그인할 수 있어야 한다.
- **FR-002**: 가입 시 워크스페이스가 자동으로 생성되며, 가입자는 Owner 역할로 설정된다.
- **FR-003**: Owner는 팀원을 이메일로 초대할 수 있으며, 초대받은 팀원은 초대 링크를 통해 가입·참여한다.
- **FR-004**: 워크스페이스 구성원 역할은 Owner와 Member로 구분된다. Owner만 워크스페이스 설정을 변경할 수 있다.

### 고객 관리

- **FR-005**: 사업자는 고객 이름, 연락처, 메모를 등록·편집·삭제할 수 있어야 한다.
- **FR-006**: 고객 목록에서 이름으로 검색할 수 있어야 한다.
- **FR-007**: 고객 상세 화면에서 해당 고객의 프로젝트 전체 목록을 확인할 수 있어야 한다.

### 프로젝트 관리

- **FR-008**: 사업자는 고객에 연결된 프로젝트를 생성할 수 있어야 한다. 프로젝트는 제목, 마감일, 담당자(선택)를 가진다.
- **FR-009**: 프로젝트 생성 시 워크플로우 템플릿(단계 목록)을 선택하여 적용할 수 있어야 한다.
- **FR-010**: 사업자는 프로젝트의 현재 단계를 변경할 수 있어야 하며, 단계 변경 이력이 타임라인에 자동으로 기록되어야 한다.
- **FR-011**: 프로젝트 상세 화면에서 수정 요청, 파일·링크, 타임라인, 메시지 생성을 단일 화면에서 처리할 수 있어야 한다.
- **FR-012**: 대시보드에서 진행 중인 프로젝트 목록, 마감 임박 프로젝트, 오늘 처리할 작업을 확인할 수 있어야 한다.
- **FR-013**: 대시보드에 "신규 접수" 목록이 있어 고객이 제출한 의뢰를 확인하고 프로젝트로 전환할 수 있어야 한다.

### 워크플로우 템플릿

- **FR-014**: 사업자는 워크스페이스 내에서 워크플로우 단계 템플릿을 생성·편집·삭제할 수 있어야 한다.
- **FR-015**: 1차 MVP에는 사진·웨딩 업종의 기본 템플릿(신규 접수 → 원본·자료 확인 → 작업자 배정 → 작업 진행 → 1차 전달 → 수정 확인 → 최종 납품 → 완료)이 기본 제공되어야 한다.
- **FR-016**: 각 단계에는 내부 표시명(사업자용)과 고객 표시명(고객용 페이지에 노출)을 별도로 설정할 수 있어야 한다.

### 수정 요청 관리

- **FR-017**: 사업자는 프로젝트에 수정 요청을 직접 등록할 수 있어야 한다. 수정 요청은 내용, 우선순위(높음/중간/낮음), 상태(접수됨/진행 중/완료), 담당자(선택)를 가진다.
- **FR-018**: 수정 요청 상태를 접수됨 → 진행 중 → 완료로 변경할 수 있어야 한다.
- **FR-019**: 고객이 고객용 페이지에서 제출한 수정 요청이 자동으로 해당 프로젝트의 수정 요청 목록에 추가되어야 한다.
- **FR-020**: 미완료 수정 요청은 대시보드 "오늘 처리할 작업"에 표시되어야 한다.

### 파일·링크 관리 (에셋)

- **FR-021**: 사업자는 프로젝트에 외부 스토리지 링크(Google Drive, Dropbox, S3 URL, NAS 등)를 에셋으로 등록할 수 있어야 한다. 에셋은 이름, URL, 타입(원본/참고/결과물/기타), 버전, 상태(준비 중/공유됨/만료됨), 만료일(선택)을 가진다.
- **FR-022**: 에셋 상태를 준비 중 → 공유됨 → 만료됨으로 변경할 수 있어야 한다.
- **FR-023**: 고객용 진행상황 페이지에는 상태가 "공유됨"인 에셋만 노출되어야 한다.

### 고객용 진행상황 페이지

- **FR-024**: 사업자는 프로젝트별 고객 공유 링크(토큰 기반)를 생성하고 활성화·비활성화할 수 있어야 한다.
- **FR-025**: 고객 공유 링크는 로그인 없이 토큰만으로 접근 가능해야 한다.
- **FR-026**: 고객용 페이지에는 현재 단계(고객 표시명), 마감 예정일, 공유된 에셋 링크 목록, 접수된 수정 요청과 완료 여부가 표시된다.
- **FR-027**: 고객용 페이지에서 재수정 접수 양식에 접근할 수 있으며, 수정 요청 내용과 파일 첨부를 제출할 수 있어야 한다.

### 고객 의뢰 접수 페이지 (인테이크)

- **FR-028**: 사업자는 워크스페이스별 신규 의뢰 접수 URL을 생성하여 고객에게 공유할 수 있어야 한다.
- **FR-029**: 신규 의뢰 접수 페이지는 로그인 없이 접근 가능하며, 고객 이름, 연락처, 의뢰 내용, 참고 파일 첨부(복수), 요청사항을 입력받는다.
- **FR-030**: 접수된 의뢰는 대시보드 "신규 접수" 목록에 표시되며, 사업자가 확인 후 고객 등록 및 정식 프로젝트 생성으로 전환할 수 있어야 한다.

### 메시지 템플릿

- **FR-031**: 사업자는 상황별 메시지 템플릿(접수 확인, 작업 시작, 납품 완료, 수정 접수 등)을 생성·편집·삭제할 수 있어야 한다. 템플릿에는 `{고객명}`, `{단계}`, `{마감일}`, `{공유링크}` 등의 변수를 사용할 수 있다.
- **FR-032**: 프로젝트 상세 화면에서 메시지 템플릿을 선택하면, 현재 프로젝트 정보가 변수에 자동으로 채워진 메시지가 클립보드에 복사된다.

## 비기능 요구사항

- **NFR-001**: 대시보드·프로젝트 목록의 초기 로딩은 3초 이내여야 한다.
- **NFR-002**: 고객용 진행상황 페이지는 모바일(375px 이상)에서 정상 열람 가능해야 한다(반응형).
- **NFR-003**: 고객용 공유 링크 토큰은 UUID v4 이상의 랜덤성을 가져야 한다(무작위 추측 불가).
- **NFR-004**: 의뢰 접수·재수정 접수 폼의 파일 첨부는 단일 파일 기준 10MB 이내로 제한한다.
- **NFR-005**: 사업자 계정 비밀번호는 bcrypt 이상의 단방향 해시로 저장되어야 하며 평문 저장이 금지된다.
- **NFR-006**: 모든 API 엔드포인트는 인증된 사용자(또는 유효한 공개 토큰)만 접근 가능해야 한다.

## 수용 기준

- **SC-001** (`FR-001`, `FR-002`): 이메일·비밀번호로 가입하면 대시보드로 이동하고, 워크스페이스가 자동으로 생성된다.
- **SC-002** (`FR-003`, `FR-004`): Owner가 팀원을 이메일로 초대하고, 초대받은 사람이 가입·참여 후 워크스페이스의 프로젝트 목록을 볼 수 있다.
- **SC-003** (`FR-005`, `FR-006`, `FR-007`): 고객을 등록하고 이름으로 검색하여 결과에서 나타나며, 해당 고객 상세 화면에서 프로젝트 목록이 표시된다.
- **SC-004** (`FR-008`, `FR-009`, `FR-010`): 고객에 연결된 프로젝트를 생성하고 워크플로우 템플릿을 적용한 후, 단계를 변경하면 타임라인에 이력이 기록된다.
- **SC-005** (`FR-014`, `FR-015`, `FR-016`): 사진·웨딩 기본 템플릿이 기본 제공되며, 사업자가 새 템플릿을 생성하고 단계별 내부명·고객 표시명을 설정할 수 있다.
- **SC-006** (`FR-017`, `FR-018`): 프로젝트에 수정 요청을 등록하고 상태를 접수됨 → 진행 중 → 완료로 변경할 수 있다.
- **SC-007** (`FR-019`, `FR-020`): 고객이 고객용 페이지에서 제출한 수정 요청이 해당 프로젝트 목록에 자동으로 추가되고, 대시보드 오늘 처리 목록에도 표시된다.
- **SC-008** (`FR-021`, `FR-022`, `FR-023`): 외부 링크를 에셋으로 등록하고 상태를 "공유됨"으로 변경하면 고객용 페이지에서 해당 링크가 보인다.
- **SC-009** (`FR-024`, `FR-025`, `FR-026`): 공유 링크를 생성하고 로그인 없이 해당 URL에 접근하면 현재 단계(고객 표시명), 마감 예정일, 공유된 에셋, 수정 요청 상태가 표시된다.
- **SC-010** (`FR-027`): 고객용 페이지에서 재수정 양식을 제출하면 해당 프로젝트에 수정 요청이 자동으로 등록된다.
- **SC-011** (`FR-028`, `FR-029`, `FR-030`): 신규 의뢰 접수 페이지(로그인 불필요)에서 의뢰를 제출하면 대시보드 "신규 접수" 목록에 나타나고, 사업자가 이를 프로젝트로 전환할 수 있다.
- **SC-012** (`FR-031`, `FR-032`): 메시지 템플릿을 생성하고, 프로젝트 상세 화면에서 해당 템플릿 선택 시 고객명·단계·공유 링크가 채워진 메시지가 클립보드에 복사된다.
- **SC-013** (`FR-012`, `FR-013`, `FR-020`): 대시보드에서 마감일이 2일 이내인 프로젝트와 미완료 수정 요청이 "오늘 처리할 작업"으로 표시된다.

## 범위 외 (Out of Scope)

- 10MB 초과 파일 직접 저장 및 영구 파일 호스팅
- 카카오톡·이메일 자동 발송
- 소셜 로그인(카카오, 구글 등)
- Owner/Member 외 세분화된 권한 체계
- AI 기능 (수정 요청 구조화, 메시지 초안 생성 등)
- 정기 결제·구독 관리 시스템
- 사진·웨딩 외 추가 업종 기본 템플릿
- 사용자 인앱 알림·이메일 알림
- 통계·리포트 대시보드

## 미결 사항 (Open Questions)

없음. 사업계획서, MVP 계획서, gohoc·ourwedding 레포 분석 결과를 바탕으로 모든 항목이 결정되었다.
