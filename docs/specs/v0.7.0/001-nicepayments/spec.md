## 배경 및 목적

Flowrit은 SaaS로 상용화하기 위해 결제 및 구독 관리 기능이 필요하다.
현재 모든 워크스페이스가 `plan = "beta"` 상태로 기능 제한 없이 운영 중이며,
상용화 전 결제 시스템을 구축하고 플랜별 기능 제한을 적용해야 한다.

국내 타겟 사용자 기반으로 **나이스페이먼츠(NICEPAY) 빌링키 기반 정기결제** 방식을 채택한다.

---

## 사용자 스토리

- **US-001**: 사용자로서, Pro 플랜에 가입하고 카드를 등록하여 매월 자동으로 결제되기를 원한다.
- **US-002**: 사용자로서, 무료 플랜에서 프로젝트 한도에 도달하면 명확한 업그레이드 안내를 받기를 원한다.
- **US-003**: 사용자로서, 연간 결제 시 할인 혜택을 받기를 원한다.
- **US-004**: 사용자로서, 구독을 취소하면 현재 결제 기간이 끝날 때까지 Pro 기능을 사용할 수 있기를 원한다.
- **US-005**: 사용자로서, 지금까지의 결제 내역을 한눈에 확인하기를 원한다.

## 기능 요구사항

### 플랜 정의 (FR-001)

**FR-001**: 시스템은 두 가지 플랜을 제공한다.

| 항목 | FREE | PRO |
|---|---|---|
| 가격 | 무료 | 월 ₩29,900 / 연 ₩298,000 |
| 활성 프로젝트 수 | 최대 3개 | 무제한 |
| 워크스페이스 추가 팀원 수 | 0명 (OWNER 본인만) | 4명 (총 5명) |
| 팀원 초대 | 불가 | 가능 |

> **기존 beta 워크스페이스**: 출시 시점에 FREE 플랜으로 전환된다.
> 기존에 생성된 3개 초과 프로젝트와 팀원은 보존되며, 새로 생성하는 것만 제한된다.

### 카드 등록 및 빌링키 발급 (FR-002)

**FR-002**: 사용자는 나이스페이먼츠 `AUTHNICE` 카드 인증 UI를 통해 카드를 등록할 수 있다.

- 카드 등록 UI: `/settings?tab=billing`의 업그레이드 모달
- 카드 인증 완료 후 클라이언트는 `authToken`과 카드 등록용 `orderId`를 서버에 전달한다.
- 서버는 나이스페이먼츠 `/v1/subscribe/regist` API를 호출하여 `bid`(빌링키)를 발급받는다.
- 발급된 빌링키는 워크스페이스의 Subscription 레코드에 저장된다.

### 첫 결제 및 구독 활성화 (FR-003)

**FR-003**: 빌링키 발급 직후 첫 결제를 즉시 실행하고, 성공 시 구독을 활성화한다.

- 결제 성공 시: `Workspace.plan = "pro"`, `Subscription.status = "active"` 로 설정
- `currentPeriodStart` = 결제 시각, `currentPeriodEnd` = 결제 주기(월/연)에 따라 계산
- 결제 실패 시 구독이 활성화되지 않고 오류 메시지를 표시한다.

### 자동결제 (FR-004)

**FR-004**: 시스템은 구독 갱신일에 자동으로 결제를 실행한다.

- Vercel Cron이 매일 실행하여 당일 만료 예정 구독을 확인하고 결제를 시도한다.
- 결제 성공 시 `currentPeriodEnd`를 다음 주기로 갱신한다.

### 결제 실패 처리 (FR-005)

**FR-005**: 자동결제 실패 시 재시도하고, 최종 실패 시 구독 상태를 변경한다.

- 결제 실패 시 최대 3일 간격으로 재시도한다 (1일차, 3일차, 7일차).
- 3회 재시도 후에도 실패 시: `Subscription.status = "past_due"`, `Workspace.plan = "free"` 로 변경.
- 결제 실패 시 워크스페이스 OWNER에게 이메일 알림을 발송한다.

### 구독 취소 (FR-006)

**FR-006**: 사용자는 언제든지 구독을 취소할 수 있다.

- 취소 시 `Subscription.cancelAtPeriodEnd = true` 로 설정.
- `currentPeriodEnd`까지 Pro 기능을 유지한다.
- 기간 만료 시 자동으로 `Workspace.plan = "free"` 로 전환된다.

### 플랜 제한 강제 (FR-007)

**FR-007**: FREE 플랜 워크스페이스에서 제한을 초과하는 액션 시 업그레이드 유도 UI를 표시한다.

- 프로젝트 3개 초과 생성 시도: 생성 차단 + 업그레이드 안내 모달
- 팀원 초대 시도: 초대 차단 + 업그레이드 안내 모달

### 청구 페이지 (FR-008)

**FR-008**: 설정 페이지에 결제 및 구독 정보를 표시하는 탭을 제공한다.

- 현재 플랜 정보 (플랜명, 다음 결제일, 결제 금액)
- 결제 수단 (등록된 카드 정보 — 마지막 4자리만 표시)
- 결제 내역 목록 (날짜, 금액, 상태)
- 업그레이드 / 취소 버튼

### 연간 결제 (FR-009)

**FR-009**: 월정기 또는 연정기 중 하나를 선택하여 구독할 수 있다.

- 연간 결제: ₩298,000 / 1회 청구 (약 17% 할인)
- `Subscription.billingCycle = "yearly"`, `currentPeriodEnd` = 1년 후

### 기존 베타 워크스페이스 마이그레이션 (FR-010)

**FR-010**: 출시 시 기존 `plan = "beta"` 워크스페이스를 FREE 플랜으로 일괄 전환한다.

- DB 마이그레이션으로 `plan = "free"` 로 업데이트
- 각 워크스페이스에 FREE Subscription 레코드 생성

## 비기능 요구사항

- **NFR-001**: 결제 금액은 서버 사이드에서만 결정한다. 클라이언트에서 전달하는 금액 파라미터를 사용하지 않는다.
- **NFR-002**: 나이스페이먼츠 `bid`는 워크스페이스별 DB에 저장한다. 카드 원문 정보는 저장하지 않는다.
- **NFR-003**: 모든 결제 API 호출 실패는 Sentry에 기록한다.
- **NFR-004**: 결제 관련 Cron 엔드포인트는 `CRON_SECRET`으로 인증한다.

## 수용 기준

- **SC-001** (`FR-007` 관련): FREE 워크스페이스에서 활성 프로젝트 3개인 상태에서 프로젝트 생성 Server Action 호출 시 에러가 반환되고 생성되지 않는다.
- **SC-002** (`FR-007` 관련): FREE 워크스페이스에서 팀원 초대 Server Action 호출 시 에러가 반환되고 초대가 생성되지 않는다.
- **SC-003** (`FR-002` 관련): 카드 등록 콜백(`/api/billing/callback`) 처리 후 나이스페이먼츠 `bid`가 `Subscription.billingKey`에 저장된다.
- **SC-004** (`FR-003` 관련): 첫 결제 성공 시 `Workspace.plan = "pro"`, `Subscription.status = "active"`, `Payment.status = "done"` 이 DB에 저장된다.
- **SC-005** (`FR-004` 관련): 자동결제 성공 시 `currentPeriodEnd`가 다음 주기로 갱신되고 새 Payment 레코드가 생성된다.
- **SC-006** (`FR-005` 관련): 나이스페이먼츠 결제 API가 실패를 반환하면 `Payment.status = "failed"`, `Payment.failReason`에 사유가 저장된다.
- **SC-007** (`FR-005` 관련): 3회 재시도 후 최종 실패 시 `Subscription.status = "past_due"`, `Workspace.plan = "free"` 로 변경된다.
- **SC-008** (`FR-006` 관련): 구독 취소 요청 시 `Subscription.cancelAtPeriodEnd = true` 로 설정되고 즉시 다운그레이드되지 않는다.
- **SC-009** (`FR-008` 관련): 결제 내역 페이지에서 해당 워크스페이스의 Payment 목록이 최신순으로 표시된다.
- **SC-010** (`FR-009` 관련): 연간 결제 선택 시 ₩298,000이 청구되고 `currentPeriodEnd`가 365일 후로 설정된다.
- **SC-011** (`FR-010` 관련): 마이그레이션 실행 후 `plan = "beta"` 워크스페이스가 0개가 된다.

## 범위 외

- 플랜 업그레이드/다운그레이드 (Monthly ↔ Yearly 전환): 추후 spec으로 분리
- 기업 플랜 / 커스텀 플랜: MVP 범위 아님
- 프로모션 코드 / 쿠폰: MVP 범위 아님
- 환불 처리 UI: 초기에는 관리자 직접 나이스페이먼츠 관리자 콘솔에서 처리
- 베타 사용자 무료 체험 기간 부여: 범위 아님 (FREE 플랜 직접 전환)
- 인보이스 발행 (세금계산서): MVP 범위 아님
