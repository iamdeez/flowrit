## 배경 및 목적

현재 Flowrit는 비밀번호 분실 시 재설정 수단이 없다. 사용자가 비밀번호를 잊으면 계정에 영구적으로 접근 불가 상태가 된다. Resend 이메일 서비스가 이미 연동되어 있으므로, 이메일 기반 비밀번호 초기화 플로우를 구현한다.

## 사용자 스토리

- **US-001**: 사용자로서, 비밀번호를 잊었을 때 이메일로 재설정 링크를 받아 새 비밀번호를 설정하고 싶다.
- **US-002**: 사용자로서, 초기화 링크가 만료되거나 이미 사용된 경우 안내 메시지를 받고 싶다.

## 기능 요구사항

- **FR-001**: 로그인 페이지에 "비밀번호 찾기" 링크를 제공한다.
- **FR-002**: `/forgot-password` 페이지에서 이메일을 입력하면 초기화 이메일을 발송한다.
- **FR-003**: 이메일에 포함된 `/reset-password/[token]` 링크로 새 비밀번호를 설정한다.
- **FR-004**: 토큰은 1시간 후 만료된다.
- **FR-005**: 토큰은 1회 사용 후 무효화된다.
- **FR-006**: 존재하지 않는 이메일로 요청해도 동일한 성공 메시지를 반환한다 (이메일 열거 방어).

## 비기능 요구사항

- **NFR-001**: 비밀번호는 bcrypt로 해시한다 (기존 register 방식 동일).
- **NFR-002**: 토큰은 `crypto.randomBytes(32)`로 생성된 hex 문자열을 사용한다.

## 수용 기준

- **SC-001** (`FR-001`): 로그인 페이지에 `/forgot-password`로 이동하는 링크가 표시된다.
- **SC-002** (`FR-002`): 유효한 이메일 입력 시 DB에 `PasswordResetToken`이 생성되고 이메일이 발송된다.
- **SC-003** (`FR-003`): 유효한 토큰으로 `/reset-password/[token]` 접근 시 새 비밀번호 입력 폼이 표시된다.
- **SC-004** (`FR-003`, `FR-004`, `FR-005`): 만료되거나 사용된 토큰으로 접근 시 오류 메시지가 표시된다.
- **SC-005** (`FR-006`): 존재하지 않는 이메일 입력 시 성공 메시지가 반환된다 (실제로는 이메일 미발송).

## 범위 외

- 소셜 로그인(OAuth) 비밀번호 초기화 — 이 프로젝트는 Credentials 방식만 사용
- 비밀번호 변경 이력 관리
- 이메일 인증(emailVerified) 연동
