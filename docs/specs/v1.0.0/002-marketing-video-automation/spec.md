# Spec: 마케팅 홍보 영상 자동 제작 플로우

> Branch: 002-marketing-video-automation | Date: 2026-06-24 | Version: v1.0.0

## 배경 및 목적

Flowrit은 실제 서비스처럼 사용할 수 있는 고객 의뢰, 프로젝트 진행, 수정 요청, 납품, 고객 포털 흐름을 갖추고 있다. 다음 단계에서는 이 제품 가치를 짧은 홍보 영상으로 반복 생산해 Instagram Reels, CapCut 편집 프로젝트, 랜딩 페이지 영상 소스로 활용할 수 있어야 한다.

본 스펙은 실제 Flowrit UI를 자동으로 조작하고 녹화하여 홍보 영상 소스 번들을 생성하는 플로우를 정의한다. 자동화는 E2E 테스트처럼 빠르게 동작하지 않고, 사람이 시연하는 듯한 속도, 마우스 움직임, 대기, 포커스 강조를 포함해야 한다.

## 목표

- 실제 제품 화면 기반의 9:16 홍보 영상 원본을 자동 생성한다.
- Flowrit의 핵심 가치인 의뢰 접수, 진행 단계 확인, 납품 이력, 수정 요청, 최종 확정 흐름을 짧은 시나리오로 보여준다.
- CapCut에서 바로 편집할 수 있는 컷별 편집 브리프와 소스 번들을 만든다.
- Instagram Reels 게시에 필요한 MP4, 커버 이미지, 캡션, 해시태그를 함께 생성한다.
- 실제 고객 데이터가 노출되지 않도록 데모 데이터와 민감 정보 검사를 강제한다.

## 범위

### MVP 범위

- Playwright 기반 홍보 영상 녹화 스크립트
- 인간적인 속도의 UI 조작 유틸
- 9:16 Reels용 기본 시나리오 3개
- 원본 녹화 MP4, 커버 이미지, 컷별 스크린샷, 캡션, 해시태그, CapCut 편집 브리프 생성
- 데모 계정/워크스페이스만 대상으로 실행하는 안전장치
- 산출물 내 이메일, 토큰, 실제 고객명 등 민감 정보 검사

### 후속 범위

- CapCut 데스크톱/웹 편집 프로젝트로 수동 가져오기
- Meta Graph API 기반 Instagram Reels 자동 게시 또는 예약
- 앱 내부에서 홍보 영상 생성 실행 이력 관리
- A/B 테스트용 카피·영상 변형 자동 생성

## 사용자 스토리

- **US-001**: 서비스 운영자로서, Flowrit의 실제 동작 화면을 사용해 Reels용 홍보 영상 소스를 자동으로 만들고 싶다.
- **US-002**: 마케터로서, 매번 수동 녹화하지 않고 같은 품질의 영상, 커버, 캡션, 해시태그를 반복 생성하고 싶다.
- **US-003**: 마케터로서, CapCut에 넣을 컷별 설명과 원본 화면 자료를 한 번에 받고 싶다.
- **US-004**: 운영자로서, 홍보 영상에 실제 고객 정보나 내부 데이터가 노출되지 않게 막고 싶다.
- **US-005**: 창업자로서, Instagram에 바로 게시할 수 있는 형태의 짧은 제품 소개 번들을 받고 싶다.

## 기능 요구사항

- **FR-001**: 시스템은 `promo:record` 명령으로 지정된 홍보 시나리오를 실행하고 브라우저 화면을 녹화해야 한다.
- **FR-002**: 녹화 자동화는 click, hover, scroll, type, wait 동작에 사람이 조작하는 듯한 속도와 pause를 적용해야 한다.
- **FR-003**: 기본 시나리오는 최소 3개를 제공해야 한다.
  - `owner-dashboard`: 작업자가 오늘의 프로젝트와 납품 상태를 확인하는 흐름
  - `client-portal`: 고객이 진행 단계와 납품 이력을 확인하는 흐름
  - `revision-to-delivery`: 수정 요청 확인 후 재납품/최종 확정으로 이어지는 흐름
- **FR-004**: 모든 시나리오는 데모 계정, 데모 워크스페이스, 데모 프로젝트만 대상으로 실행되어야 한다.
- **FR-005**: 시스템은 실행 전 `PROMO_BASE_URL`, `PROMO_EMAIL`, `PROMO_PASSWORD`, `PROMO_PUBLIC_TOKEN` 등 필수 환경값을 검증해야 한다.
- **FR-006**: 시스템은 결과물 번들을 `artifacts/promo/{runId}/` 또는 설정된 출력 경로에 생성해야 한다.
- **FR-007**: 결과물 번들은 원본 MP4, 커버 이미지, 컷별 스크린샷, `manifest.json`, `caption.md`, `capcut-brief.md`를 포함해야 한다.
- **FR-008**: `manifest.json`은 시나리오명, 실행 시간, base URL, viewport, 산출 파일 목록, 사용한 데모 프로젝트 식별자를 포함해야 한다.
- **FR-009**: 시스템은 산출물 텍스트와 manifest에 이메일, 토큰, secret, 실제 고객명 blacklist 등 민감 정보가 포함되는지 검사해야 한다.
- **FR-010**: CapCut 계정이나 유료 템플릿 없이도 수동 편집 가능한 handoff 패키지는 생성되어야 한다.
- **FR-011**: Instagram 자동 게시 권한이 없을 때도 Reels 수동 게시용 MP4, 커버, 캡션, 해시태그는 생성되어야 한다.
- **FR-012**: 자동 게시 기능을 도입하는 경우 Meta OAuth token, page id, instagram business account id는 서버 환경변수 또는 보안 저장소로만 관리해야 한다.
- **FR-013**: 녹화 스크립트 실패 시 실패 장면 스크린샷과 로그를 남겨 재촬영 원인을 파악할 수 있어야 한다.
- **FR-014**: 생성된 영상은 기본 9:16 세로 비율을 만족해야 하며, 모바일 첫 화면에서 텍스트가 잘리지 않아야 한다.
- **FR-015**: 운영자는 생성된 번들을 게시 전 검토할 수 있어야 하며, 자동 게시가 활성화되어도 즉시 게시와 수동 검토 모드를 선택할 수 있어야 한다.

## 비기능 요구사항

- **NFR-001**: 실제 고객 데이터, 결제 정보, 내부 메모, 비공개 파일 URL은 영상과 메타데이터에 포함되지 않아야 한다.
- **NFR-002**: 스크립트는 production 실데이터가 아닌 demo, preview, staging 성격의 환경을 대상으로 실행해야 한다.
- **NFR-003**: Playwright selector는 가능한 접근성 label, role, text를 우선 사용해 UI 구조 변경에 덜 취약해야 한다.
- **NFR-004**: 결과물 생성은 로컬 또는 CI 작업 환경에서 실행하며 Vercel serverless runtime에 의존하지 않는다.
- **NFR-005**: ffmpeg 등 로컬 바이너리가 필요한 단계는 사전 검사로 누락 여부를 명확히 알려야 한다.
- **NFR-006**: 자동 게시 단계는 외부 API rate limit과 실패 응답을 재시도 없이 사용자에게 명확히 보고해야 한다.
- **NFR-007**: 영상 시나리오는 15초, 30초, 45초 버전으로 확장 가능해야 한다.

## 수용 기준

- **SC-001** (`FR-001`, `FR-002`): `npm run promo:record -- --scenario owner-dashboard` 실행 시 자연스러운 속도의 녹화 파일이 생성된다.
- **SC-002** (`FR-003`): 기본 시나리오 3개의 스크립트가 존재하고 각 시나리오가 독립 실행 가능하다.
- **SC-003** (`FR-004`, `FR-005`): 필수 `PROMO_*` 값이 없거나 데모 workspace 검증에 실패하면 녹화를 시작하지 않는다.
- **SC-004** (`FR-006`, `FR-007`, `FR-008`): 실행 후 출력 폴더에 MP4, cover PNG, 스크린샷, manifest, caption, CapCut edit brief가 생성된다.
- **SC-005** (`FR-009`, `NFR-001`): 산출물 검사에서 민감 정보 패턴이 발견되면 명령이 실패하고 문제 위치를 보고한다.
- **SC-006** (`FR-010`): CapCut 유료 기능 없이도 수동 편집 가능한 edit brief와 source bundle이 생성된다.
- **SC-007** (`FR-011`): Instagram 권한 없이도 수동 게시 가능한 caption과 hashtag 문서가 생성된다.
- **SC-008** (`FR-013`): 실패 시 screenshot과 로그가 남아 어느 단계에서 실패했는지 알 수 있다.
- **SC-009** (`FR-014`): 기본 산출 영상은 9:16 비율이며 390px 모바일 기준 주요 텍스트가 잘리지 않는다.
- **SC-010** (`FR-015`): 자동 게시가 도입된 경우에도 preview/review-only 모드가 기본값이다.

## 범위 외

- 광고 예산 집행, 캠페인 성과 분석, Meta Ads Manager 자동화
- 유료 음악, 음성, 모델 초상권 라이선스 구매 대행
- CapCut 내부 영상 편집 결과의 품질 보장
- Instagram 앱 심사, 비즈니스 인증, 계정 제재 대응
- 실제 고객 프로젝트를 홍보 영상으로 자동 변환하는 기능

## 사용자가 준비해야 하는 사항

- **브랜드 자료**: 로고, 대표 색상, 제품 한 줄 소개, CTA URL, Instagram 계정명
- **홍보 메시지**: 타깃 고객, 핵심 문제, 강조하고 싶은 기능 3개, 원하는 톤앤매너
- **데모 데이터**: 공개 가능한 고객명, 프로젝트명, 납품 파일명, 수정 요청 예시, 완료 상태 예시
- **데모 계정**: 촬영 전용 Flowrit 계정, workspace, 프로젝트, 고객 포털 token
- **CapCut 준비 정보**: 데스크톱 앱 또는 웹 편집 환경, 무료 템플릿 사용 여부, 내보내기 해상도 기준
- **Instagram/Meta 준비**: Professional Instagram 계정, 연결된 Facebook Page, Meta Business, Meta App, 게시 권한
- **라이선스**: 배경 음악, 폰트, 이미지, AI 생성 영상 활용 동의 및 표시 정책
- **운영 기준**: 자동 게시까지 할지, 수동 검토 후 게시할지, 게시 빈도와 승인 담당자

## 미결 사항

없음. CapCut은 자동 생성 연동이 아니라 무료 편집 handoff 대상으로 정의한다. Instagram 자동 게시 기능은 후속 범위로 분리하고, MVP는 게시 가능한 소스 번들 자동 생성으로 정의한다.
