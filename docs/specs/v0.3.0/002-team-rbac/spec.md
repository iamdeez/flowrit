## 배경 및 목적

현재 팀 관리는 초대와 초대 취소만 가능하다. 가입 후 역할은 OWNER / MEMBER 두 가지뿐이며, Owner가 멤버의 역할을 변경하거나 멤버를 제거하는 UI가 존재하지 않는다. 팀이 3인 이상으로 성장하면 팀장(중간 관리자) 역할이 필요하고, 잘못 초대된 멤버를 제거할 수도 없다.

v0.3.0에서 ADMIN 역할을 추가하고, Owner가 멤버 역할을 변경하고 팀을 재구성할 수 있도록 한다.

### 역할 정의

| 역할 | 설명 |
|---|---|
| OWNER | 워크스페이스 최고 책임자. 모든 권한 보유. 1명만 존재. |
| ADMIN | 팀 초대·제거 가능, 워크스페이스 설정 읽기 전용. 소유권 이전·워크스페이스 삭제 불가. |
| MEMBER | 기본 역할. 고객·프로젝트·수정 요청·에셋 전체에 접근 가능. 팀 관리 불가. |

## 사용자 스토리

- **US-301**: Owner로서, 특정 멤버를 Admin으로 승격하거나 Member로 강등할 수 있기를 원한다.
- **US-302**: Owner·Admin으로서, 워크스페이스 멤버를 제거할 수 있기를 원한다.
- **US-303**: Owner로서, 워크스페이스 소유권을 다른 멤버에게 이전할 수 있기를 원한다.
- **US-304**: 팀원으로서, 내 역할이 무엇인지 팀 페이지에서 확인할 수 있기를 원한다.
- **US-305**: MEMBER 역할의 사용자로서, 내가 수행할 수 없는 동작(팀 관리, 워크스페이스 설정 변경)이 UI에서 명확하게 비활성화되기를 원한다.

## 기능 요구사항

### 스키마 변경

- **FR-301**: `WorkspaceMember.role` 필드에 `"ADMIN"` 값을 허용한다. 기존 값(`"OWNER"`, `"MEMBER"`)은 유지된다.

### 역할 변경

- **FR-302**: Owner는 팀 페이지에서 각 멤버의 역할을 변경할 수 있어야 한다.
  - MEMBER → ADMIN 승격
  - ADMIN → MEMBER 강등
  - Owner 자신의 역할은 변경할 수 없다.
- **FR-303**: 역할 변경 시 별도 확인 대화상자 없이 즉시 적용되며, 성공 시 목록이 갱신된다.

### 멤버 제거

- **FR-304**: Owner 또는 Admin은 팀 페이지에서 멤버를 제거할 수 있어야 한다.
  - Admin은 MEMBER만 제거 가능. 다른 ADMIN·OWNER는 제거 불가.
  - Owner는 모든 MEMBER·ADMIN을 제거 가능.
  - 자기 자신은 제거 불가(이 경우 001-settings의 "워크스페이스 나가기" 사용).
- **FR-305**: 멤버 제거 시 확인 대화상자를 통해 이름·이메일을 표시하고 재확인한다.
- **FR-306**: 제거된 멤버가 프로젝트의 `assigneeId`인 경우, 해당 프로젝트의 담당자가 `null`로 설정된다.

### 소유권 이전

- **FR-307**: Owner는 현재 ADMIN 또는 MEMBER 역할의 멤버에게 소유권을 이전할 수 있어야 한다.
  - 이전 후 기존 Owner는 ADMIN 역할로 강등된다.
  - 확인 대화상자에서 이전 대상자의 이름과 이메일을 표시한다.
- **FR-308**: 소유권 이전 후에도 이전 Owner는 워크스페이스에 계속 잔류한다(강퇴되지 않음).

### UI 권한 제어

- **FR-309**: 팀 페이지에서 현재 사용자의 역할에 따라 UI를 조건부로 표시한다.
  - MEMBER: 팀원 목록과 보류 중인 초대만 읽기 전용 표시. 초대 폼, 역할 변경, 제거 버튼 숨김.
  - ADMIN: 초대 폼, MEMBER 제거 버튼, 초대 취소 버튼 표시. 역할 변경 및 소유권 이전 버튼 숨김.
  - OWNER: 모든 기능 표시.

## 비기능 요구사항

- **NFR-301**: 역할 변경·멤버 제거·소유권 이전은 Server Action에서 현재 사용자의 역할을 DB에서 재확인한다. 클라이언트 단 권한 우회를 방지한다.
- **NFR-302**: Admin이 OWNER·ADMIN 제거를 시도하는 Server Action 호출은 403으로 거부한다.
- **NFR-303**: 소유권 이전은 단일 트랜잭션으로 처리한다(기존 Owner ADMIN 강등 + 신규 Owner 승격).

## 수용 기준

- **SC-301** (`FR-302`): Owner 계정으로 로그인 후 팀 페이지에서 MEMBER를 ADMIN으로 변경하면 해당 멤버의 배지가 "어드민"으로 바뀐다.
- **SC-302** (`FR-304`, `FR-305`): Owner가 멤버 제거 확인 후 제거를 실행하면 해당 멤버가 목록에서 사라진다. 제거된 멤버가 로그인 시 해당 워크스페이스에 접근 불가 상태가 된다.
- **SC-303** (`FR-304`): Admin 계정으로 ADMIN 역할의 다른 멤버 제거를 시도하면 거부된다.
- **SC-304** (`FR-306`): 제거된 멤버가 담당자였던 프로젝트에서 담당자가 "미지정"으로 표시된다.
- **SC-305** (`FR-307`, `FR-308`): 소유권 이전 후, 기존 Owner가 다시 팀 페이지를 보면 자신의 역할이 "어드민"으로 표시된다. 새 Owner는 소유권 이전 및 워크스페이스 삭제 버튼이 활성화된다.
- **SC-306** (`FR-309`): MEMBER 계정으로 팀 페이지 접근 시 초대 폼과 역할 변경 컨트롤이 표시되지 않는다.
