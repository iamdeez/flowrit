## 배경 및 목적

현재 Flowrit에서 생성된 데이터(고객, 프로젝트, 수정 요청 이력)를 외부로 내보낼 방법이 없다. 또한 완료된 프로젝트가 목록에 계속 남아 있어 운영 중인 프로젝트를 찾기 어렵고, 같은 구조의 프로젝트를 반복 생성할 때 처음부터 입력해야 한다.

이 스펙은 (1) CSV 데이터 내보내기, (2) 프로젝트 아카이브, (3) 프로젝트 복제를 통해 데이터 생명주기를 체계화한다.

---

## 사용자 스토리

- **US-601**: 사업자로서, 고객 목록을 CSV로 내보내 세금계산서 발행 등 외부 작업에 활용하고 싶다.
- **US-602**: 사업자로서, 프로젝트 내역을 CSV로 내보내 월말 정산이나 포트폴리오 관리에 사용하고 싶다.
- **US-603**: 사업자로서, 완료된 프로젝트를 아카이브하여 활성 프로젝트 목록을 깔끔하게 유지하고 싶다.
- **US-604**: 사업자로서, 아카이브된 프로젝트도 필요 시 다시 확인할 수 있기를 원한다.
- **US-605**: 사업자로서, 정기적으로 진행하는 동일 유형의 프로젝트를 기존 프로젝트를 복제하여 빠르게 생성하고 싶다.

## 기능 요구사항

### CSV 내보내기

- **FR-601**: 고객 목록 페이지(`/customers`)에 "CSV 내보내기" 버튼을 추가한다. 클릭 시 현재 워크스페이스의 고객 전체를 CSV 파일로 다운로드한다.
  - 내보내기 컬럼: 이름, 연락처, 메모, 프로젝트 수, 등록일
- **FR-602**: 프로젝트 목록 페이지(`/projects`)에 "CSV 내보내기" 버튼을 추가한다. 현재 필터(검색어·상태·아카이브 여부)가 적용된 결과를 내보낸다.
  - 내보내기 컬럼: 프로젝트명, 고객명, 현재 단계, 마감일, 담당자, 예산, 완료 여부, 생성일
- **FR-603**: CSV 인코딩은 UTF-8 BOM(`﻿`)을 포함하여 Excel에서 한글이 정상 표시되도록 한다.
- **FR-604**: 내보내기는 서버 Route Handler(`/api/export/customers`, `/api/export/projects`)로 구현한다. 해당 워크스페이스 멤버만 접근 가능하다.

### 프로젝트 아카이브

- **FR-605**: `Project` 모델에 `archivedAt DateTime?` 필드를 추가한다. `archivedAt`이 null이 아닌 프로젝트를 "아카이브됨"으로 간주한다.
- **FR-606**: 프로젝트 상세 페이지에 "아카이브" 버튼을 추가한다. 클릭 시 확인 없이 즉시 아카이브된다. 아카이브된 프로젝트에는 "아카이브 해제" 버튼이 표시된다.
- **FR-607**: 프로젝트 목록 페이지의 상태 필터에 "아카이브됨" 항목을 추가한다. 기본 필터("전체", "진행 중", "완료")에서는 아카이브된 프로젝트가 표시되지 않는다.
- **FR-608**: 아카이브 시 타임라인에 `ARCHIVED` 이벤트가 기록된다. 아카이브 해제 시 `UNARCHIVED` 이벤트가 기록된다.
- **FR-609**: 아카이브된 프로젝트는 대시보드의 "진행 중" 카운트, "마감 임박", "오늘 처리할 작업"에서 제외된다.

### 프로젝트 복제

- **FR-610**: 프로젝트 상세 페이지에 "복제" 버튼을 추가한다.
- **FR-611**: 복제 대화상자에서 새 프로젝트의 제목, 고객(선택), 마감일(선택)을 입력한다. 제목 기본값은 `[원본 제목] (복사본)`.
- **FR-612**: 복제 시 다음 항목이 새 프로젝트로 복사된다.
  - 워크플로우 단계 목록(이름·고객표시명·순서, `completedAt`은 null로 초기화)
  - `currentStageId`는 복제된 첫 번째 단계로 설정
- **FR-613**: 복제 시 다음 항목은 복사되지 않는다: 수정 요청, 에셋, 타임라인 이벤트, 공유 링크, `assigneeId`.
- **FR-614**: 복제 완료 후 새로 생성된 프로젝트 상세 페이지로 이동한다.

## 비기능 요구사항

- **NFR-601**: CSV 내보내기 Route Handler는 세션 인증과 워크스페이스 소속 검증을 수행한다. 인증되지 않은 요청은 401, 타 워크스페이스 접근은 403으로 응답한다.
- **NFR-602**: 대규모 데이터 내보내기(예: 고객 1,000명)에서도 메모리 초과 없이 스트리밍으로 응답한다. Prisma `cursor` 기반 페이지네이션 또는 Node.js `Readable` 스트림을 활용한다.
- **NFR-603**: 프로젝트 복제는 단일 트랜잭션으로 처리한다(프로젝트 생성 + 단계 일괄 생성).

## 수용 기준

- **SC-601** (`FR-601`, `FR-603`): 고객 목록 페이지에서 "CSV 내보내기"를 클릭하면 `customers.csv` 파일이 다운로드되며, Excel에서 열었을 때 한글이 깨지지 않는다.
- **SC-602** (`FR-602`): 프로젝트 목록에서 "진행 중" 필터 적용 후 CSV를 내보내면 진행 중인 프로젝트만 포함된다.
- **SC-603** (`FR-605`, `FR-606`, `FR-607`): 프로젝트를 아카이브하면 기본 "전체" 목록에서 사라지고, "아카이브됨" 필터 선택 시 다시 표시된다.
- **SC-604** (`FR-609`): 아카이브된 프로젝트는 대시보드 카운트와 마감 임박 목록에 포함되지 않는다.
- **SC-605** (`FR-610`, `FR-611`, `FR-612`, `FR-613`): 프로젝트 복제 후 새 프로젝트 상세 페이지에서 워크플로우 단계가 원본과 동일하게 존재하고, 수정 요청·에셋·타임라인은 비어 있다.
- **SC-606** (`FR-604`): 로그인하지 않은 상태에서 `/api/export/customers`에 접근하면 401이 반환된다.

## 범위 외

- 고객 CSV 가져오기(Import)
- 프로젝트 PDF 보고서 생성
- 에셋·수정 요청 이력 CSV 내보내기
- 자동 아카이브(완료 후 N일 경과 시)
- 워크스페이스 전체 데이터 백업(zip) 다운로드
- 데이터 보존 정책 설정
