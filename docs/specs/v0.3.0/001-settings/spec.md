## 배경 및 목적

사이드바 내비게이션에 "설정" 링크(`/settings`)가 이미 노출되어 있으나 해당 라우트가 존재하지 않아 클릭 시 404가 발생한다. 또한 현재 사용자는 자신의 이름·이메일·비밀번호를 변경할 방법이 없고, 워크스페이스 Owner도 워크스페이스 이름·슬러그를 변경할 수 없다.

v0.3.0의 첫 번째 목표는 이 설정 공백을 채워 사용자가 자신의 계정과 워크스페이스를 직접 관리할 수 있도록 하는 것이다. 설정 페이지는 이후 알림, 청구, 보안 등 엔터프라이즈 기능의 허브가 된다.

---

## 사용자 스토리

- **US-201**: 사업자로서, 내 이름과 이메일을 직접 수정하고 싶다.
- **US-202**: 사업자로서, 비밀번호를 현재 비밀번호 확인 후 변경하고 싶다.
- **US-203**: 사업자로서, 워크스페이스 이름을 변경하고 싶다 (Owner만).
- **US-204**: 사업자로서, 고객 의뢰 접수 URL에 쓰이는 워크스페이스 슬러그를 변경하고 싶다 (Owner만).
- **US-205**: 사업자로서, 워크스페이스를 삭제하거나 멤버십을 직접 나갈 수 있기를 원한다.
- **US-206**: 사업자로서, 이메일 알림을 이벤트 유형별로 켜거나 끄고 싶다.

## 기능 요구사항

### 설정 페이지 라우팅

- **FR-201**: `/settings` 경로에 설정 페이지가 존재해야 한다. 사이드바의 "설정" 링크를 클릭하면 이 페이지로 이동한다.
- **FR-202**: 설정 페이지는 "프로필", "워크스페이스", "알림" 세 개의 탭으로 구성된다. 탭은 URL 파라미터(`?tab=`)로 상태를 유지한다.

### 프로필 설정

- **FR-203**: 사용자는 자신의 이름을 수정하고 저장할 수 있어야 한다. 이름은 1자 이상이어야 한다.
- **FR-204**: 사용자는 자신의 이메일 주소를 수정할 수 있어야 한다. 변경하려는 이메일이 이미 다른 계정에 등록된 경우 오류 메시지를 표시한다.
- **FR-205**: 사용자는 현재 비밀번호를 입력한 후 새 비밀번호(최소 8자)로 변경할 수 있어야 한다. 현재 비밀번호가 일치하지 않으면 오류 메시지를 표시한다.

### 워크스페이스 설정 (Owner 전용)

- **FR-206**: Owner는 워크스페이스 이름을 수정하고 저장할 수 있어야 한다.
- **FR-207**: Owner는 워크스페이스 슬러그를 수정할 수 있어야 한다. 슬러그는 영문 소문자·숫자·하이픈만 허용하며, 다른 워크스페이스와 중복될 수 없다. 슬러그 변경 시 기존 의뢰 접수 URL(`/intake/[slug]`)이 달라짐을 경고로 안내한다.
- **FR-208**: Member·Admin 역할의 사용자에게는 워크스페이스 탭이 읽기 전용으로 표시된다(이름·슬러그 수정 불가).

### 알림 설정

- **FR-209**: 사용자는 다음 이메일 알림을 개별적으로 켜거나 끌 수 있어야 한다.
  - `notify_new_inquiry`: 신규 의뢰가 접수되었을 때
  - `notify_revision_submitted`: 고객 포털에서 수정 요청이 제출되었을 때
  - `notify_stage_changed`: 담당 프로젝트의 단계가 변경되었을 때
  - `notify_deadline_soon`: 마감일 1일 전 리마인더
- **FR-210**: 알림 설정 값은 사용자 단위로 저장된다. `User` 모델에 `notificationSettings Json @default("{}")` 필드를 추가한다.

### 위험 구역 (Danger Zone)

- **FR-211**: 멤버(Member·Admin)는 "워크스페이스 나가기" 버튼을 통해 해당 워크스페이스에서 자신을 제거할 수 있다. 확인 대화상자를 거친다.
- **FR-212**: Owner는 "워크스페이스 삭제" 버튼을 통해 워크스페이스와 모든 하위 데이터를 영구 삭제할 수 있다. 삭제 전 워크스페이스 이름을 직접 입력하여 확인한다.
- **FR-213**: Owner가 유일한 멤버인 경우에만 삭제가 허용된다. 다른 멤버가 있는 상태에서 삭제를 시도하면 "모든 팀원을 먼저 제거하거나 소유권을 이전하세요"라는 오류를 표시한다.

## 비기능 요구사항

- **NFR-201**: 비밀번호 변경 처리는 bcrypt 재해싱으로 수행한다. 새 비밀번호는 응답 본문에 포함되지 않는다.
- **NFR-202**: 워크스페이스 탭은 현재 사용자의 역할을 서버에서 확인하여 렌더링한다. 클라이언트 단 역할 체크만으로 수정을 제한하지 않는다.
- **NFR-203**: 슬러그 중복 검사는 서버 Action 내에서 단일 트랜잭션으로 처리한다.

## 수용 기준

- **SC-201** (`FR-201`, `FR-202`): 사이드바 "설정" 클릭 시 `/settings` 페이지가 열리고, 프로필·워크스페이스·알림 탭이 표시된다.
- **SC-202** (`FR-203`): 이름을 수정하고 저장하면 사이드바 등 레이아웃의 사용자 이름이 갱신된다.
- **SC-203** (`FR-204`): 이미 사용 중인 이메일로 변경 시도 시 오류 메시지가 표시되고 저장되지 않는다.
- **SC-204** (`FR-205`): 현재 비밀번호가 틀리면 오류를 표시한다. 정상 변경 후 로그아웃 없이 계속 사용 가능하다.
- **SC-205** (`FR-206`, `FR-207`): Owner가 슬러그를 변경하면 `/intake/[새슬러그]` URL이 즉시 유효해지고 기존 슬러그는 더 이상 동작하지 않는다.
- **SC-206** (`FR-208`): Member로 로그인한 사용자는 워크스페이스 탭에서 이름·슬러그 입력이 비활성화된다.
- **SC-207** (`FR-209`, `FR-210`): 특정 이메일 알림을 끄면 해당 이벤트가 발생해도 이메일이 발송되지 않는다.
- **SC-208** (`FR-211`): 멤버가 "워크스페이스 나가기"를 실행하면 해당 워크스페이스 접근이 차단되고 대시보드 접근 시 다른 워크스페이스로 리다이렉트(없으면 로그인 페이지)된다.
- **SC-209** (`FR-212`, `FR-213`): 다른 멤버가 있는 상태에서 워크스페이스 삭제 시도 시 오류가 표시된다. 단독 Owner가 워크스페이스 이름을 입력하고 확인하면 워크스페이스와 모든 데이터가 삭제된다.

## 범위 외

- 소셜 계정 연동(Google, 카카오)
- 프로필 이미지(아바타) 업로드
- 다중 워크스페이스 전환 UI
- 2단계 인증(2FA)
- 워크스페이스 로고 업로드
- API 키 발급 관리

## 미결 사항

없음. 현재 코드베이스(User·Workspace·WorkspaceMember 스키마)와 기존 인증 흐름을 분석하여 모든 항목이 결정되었다.
