## 업그레이드 배경

v0.1.0(MVP)과 v0.2.0(UX 개선)을 통해 핵심 업무 흐름(의뢰 접수 → 프로젝트 관리 → 고객 공유)이 완성되었다. v0.3.0은 1~5인 파일럿 사용자를 넘어 **실제 팀 단위 유료 고객**을 수용하기 위한 엔터프라이즈 기반 기능을 갖춘다.

## 현재 상태 진단

| 영역 | 현재 상태 | 문제 |
|---|---|---|
| 설정 | `/settings` 사이드바 링크 존재 | 404 발생 — 라우트 없음 |
| 프로필 관리 | 없음 | 이름·이메일·비밀번호 변경 불가 |
| 팀 역할 | OWNER / MEMBER만 존재 | ADMIN 역할 없음, 멤버 제거 불가, 역할 변경 불가 |
| 알림 | 초대 이메일만 존재 | 비즈니스 이벤트(의뢰 접수, 수정 요청 등) 알림 없음 |
| 통계 | 숫자 2개(진행 중, 마감 임박) | 비즈니스 인사이트 전무 |
| 데이터 관리 | 없음 | CSV 내보내기 없음, 프로젝트 아카이브 없음, 복제 없음 |

## v0.3.0 스펙 구성

| 스펙 | 경로 | 핵심 기능 | 우선순위 |
|---|---|---|---|
| **001-settings** | `/settings` 페이지 신설 | 프로필 수정, 워크스페이스 설정, 알림 설정, 위험 구역 | P0 — 네비 링크 404 수정 |
| **002-team-rbac** | `/team` 개선 | ADMIN 역할, 역할 변경, 멤버 제거, 소유권 이전 | P1 — 팀 성장 필수 |
| **003-notifications** | 이메일 + 인앱 알림 | 의뢰·수정·단계·마감 이메일, 알림 벨 | P1 — 이탈 방지 핵심 |
| **004-analytics** | `/analytics` 신설 | 완료 추이, 수정 분석, 팀 워크로드, 예산 관리 | P2 — 데이터 기반 운영 |
| **005-data-management** | 기존 화면에 기능 추가 | CSV 내보내기, 프로젝트 아카이브, 프로젝트 복제 | P2 — 데이터 생명주기 |

## 구현 우선순위

### P0 — 즉시 수정 (MVP 버그)

- **001-settings**: `/settings` 404 수정. 사이드바에 링크가 있는 페이지가 없는 것은 제품 신뢰도를 크게 저하시킨다.

### P1 — 팀 성장 필수 기능

- **002-team-rbac**: 팀이 3인 이상 되는 순간 ADMIN 역할과 멤버 제거가 필요해진다.
- **003-notifications**: 사용자가 앱을 열지 않아도 중요 이벤트를 인지할 수 있어야 이탈이 줄어든다.

### P2 — 운영 성숙도 향상

- **004-analytics**: 월 10건 이상 프로젝트를 운영하는 팀에서 데이터 기반 의사결정이 필요해진다.
- **005-data-management**: 외부 정산·보고를 위한 내보내기와 목록 관리(아카이브) 없이는 운영이 지저분해진다.

## 스키마 변경 요약

| 모델 | 변경 유형 | 변경 내용 |
|---|---|---|
| `User` | 필드 추가 | `notificationSettings Json @default("{}")` |
| `Project` | 필드 추가 | `budget Int?`, `archivedAt DateTime?` |
| `Notification` | 신규 모델 | 인앱 알림 저장 (FR-410) |
| `WorkspaceMember.role` | 값 추가 | `"ADMIN"` (기존 OWNER·MEMBER 유지) |
| `TimelineEvent.eventType` | 값 추가 | `ARCHIVED`, `UNARCHIVED`, `DEADLINE_NOTIFIED` |
