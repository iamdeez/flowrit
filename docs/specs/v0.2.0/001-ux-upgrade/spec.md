## 배경 및 목적

v0.1.0 MVP가 완성된 후 실제 업무 흐름을 검토한 결과, 다음 마찰점이 확인되었다.

**흐름의 단절**
- 고객 상세 → "새 프로젝트" 클릭 시 고객 선택이 초기화되어 다시 선택해야 한다.
- 신규 의뢰를 프로젝트로 전환할 때 마감일을 입력하는 단계가 없다.
- 공유 링크 텍스트가 화면에 표시되지만 복사 버튼이 없어 직접 드래그 선택해야 한다.

**정보 가시성 부족**
- 프로젝트 상세에서 현재 단계를 드롭다운 텍스트로만 표시하여 전체 워크플로우 중 어디에 있는지 파악하기 어렵다.
- 대시보드에 워크스페이스 전체의 진행 현황 요약이 없어 전체 업무량을 한눈에 파악하기 어렵다.
- 프로젝트 목록에 검색 기능이 없어 프로젝트가 늘어날수록 찾기 어렵다.

**누락된 기능**
- 에셋 삭제 기능이 없어 잘못 등록한 에셋을 제거할 수 없다.
- 수정 요청 삭제 기능이 없어 잘못 접수된 수정 요청이 목록에 남는다.
- 타임라인이 자동 이벤트만 기록되어 사업자가 내부 메모를 추가할 수 없다.
- 고객 포털에서 파일을 첨부하여 수정 요청 제출 시, 파일 URL이 텍스트 본문에 이어붙여져 사업자가 파일과 내용을 구분하기 어렵다.

v0.2.0은 이 마찰점들을 해소하여 핵심 업무 흐름(의뢰 접수 → 프로젝트 관리 → 고객 공유)이 끊김 없이 이어지도록 한다.

## 사용자 스토리

### 사업자 (Service Provider)

- **US-101**: 사업자로서, 고객 상세 화면에서 바로 새 프로젝트를 생성할 때 해당 고객이 자동으로 선택되어 있기를 원한다.
- **US-102**: 사업자로서, 프로젝트 목록에서 제목이나 고객명으로 검색하여 원하는 프로젝트를 빠르게 찾고 싶다.
- **US-103**: 사업자로서, 프로젝트 상세 화면에서 현재 단계가 전체 워크플로우 중 어디에 있는지 시각적으로 한눈에 파악하고 싶다.
- **US-104**: 사업자로서, 타임라인에 내부 메모를 남겨 후속 작업 담당자에게 맥락을 전달하고 싶다.
- **US-105**: 사업자로서, 고객이 고객 포털에서 파일을 첨부하여 제출한 수정 요청에서 파일 링크가 본문과 분리되어 표시되기를 원한다.
- **US-106**: 사업자로서, 실수로 등록한 에셋을 삭제할 수 있기를 원한다.
- **US-107**: 사업자로서, 잘못 접수된 수정 요청을 삭제할 수 있기를 원한다.
- **US-108**: 사업자로서, 공유 링크를 버튼 하나로 클립보드에 복사하고 싶다.
- **US-109**: 사업자로서, 대시보드에서 워크스페이스 전체 진행 현황(진행 중 프로젝트 수, 마감 임박 수)을 한눈에 파악하고 싶다.
- **US-110**: 사업자로서, 신규 의뢰를 프로젝트로 전환할 때 마감일을 바로 입력하고 싶다.

## 기능 요구사항

### 프로젝트 생성 흐름 개선

- **FR-101**: 고객 상세 화면의 "새 프로젝트" 버튼은 프로젝트 생성 폼을 `?customerId={id}` 파라미터와 함께 열어야 한다. 프로젝트 생성 폼은 이 파라미터가 있으면 해당 고객을 미리 선택된 상태로 표시해야 한다.
- **FR-102**: 신규 의뢰 → 프로젝트 전환 대화상자에 마감일(선택) 입력 필드를 추가한다.

### 프로젝트 목록 검색

- **FR-103**: 프로젝트 목록 화면에 검색 입력창을 추가한다. 검색어는 프로젝트 제목 또는 고객명에 대해 대소문자 무관 부분 일치로 적용된다. 검색 결과는 기존 상태 필터(전체/진행 중/완료)와 함께 동작한다.

### 단계 시각화

- **FR-104**: 프로젝트 상세 헤더의 현재 단계 표시 영역을 시각적 스테퍼로 교체한다. 스테퍼는 프로젝트의 전체 단계를 순서대로 나열하며, 현재 단계 이전의 단계는 완료 표시, 현재 단계는 강조, 이후 단계는 비활성 표시를 한다. 단계 클릭으로 단계를 변경할 수 있어야 한다.

### 타임라인 메모

- **FR-105**: 프로젝트 상세의 타임라인 탭에서 사업자가 내부 메모 텍스트를 입력·저장할 수 있어야 한다. 저장된 메모는 `MEMO` 이벤트 타입으로 타임라인에 시간순으로 기록된다.

### 수정 요청 파일 분리

- **FR-106**: `RevisionRequest` 모델에 `fileUrls Json @default("[]")` 필드를 추가한다. 고객 포털에서 파일을 첨부하여 수정 요청 제출 시 파일 URL이 `content`와 별도로 `fileUrls`에 배열로 저장된다.
- **FR-107**: 사업자의 프로젝트 상세 수정 요청 목록에서 `fileUrls` 배열의 항목이 "첨부 파일" 섹션으로 분리되어 링크 형태로 표시된다.

### 에셋 삭제

- **FR-108**: 사업자는 프로젝트 상세의 에셋 목록에서 에셋을 삭제할 수 있어야 한다. 삭제 시 확인 없이 즉시 삭제되며, 타임라인에 삭제 이벤트가 기록된다.

### 수정 요청 삭제

- **FR-109**: 사업자는 프로젝트 상세의 수정 요청 목록에서 수정 요청을 삭제할 수 있어야 한다. 삭제는 상태에 관계없이 가능하다.

### 공유 링크 복사

- **FR-110**: 프로젝트 상세 헤더에서 공유 링크가 활성화된 경우, 공유 링크를 클립보드에 복사하는 버튼을 제공한다. 복사 성공 시 버튼 텍스트가 일시적으로 "복사됨"으로 변경된다.

### 대시보드 현황 요약

- **FR-111**: 대시보드 상단에 진행 현황 요약 섹션을 추가한다. 이 섹션은 진행 중인 프로젝트 전체 수와 마감이 2일 이내인 프로젝트 수를 카드 형태로 표시한다.

## 비기능 요구사항

- **NFR-101**: FR-103의 프로젝트 검색 쿼리는 서버 사이드에서 처리한다. 검색 결과 표시까지의 응답 시간은 2초 이내여야 한다.
- **NFR-102**: FR-108의 에셋 삭제, FR-109의 수정 요청 삭제는 해당 프로젝트와 같은 워크스페이스 구성원만 수행할 수 있어야 한다. 타 워크스페이스의 에셋·수정 요청을 삭제할 수 없어야 한다.
- **NFR-103**: FR-110의 공유 링크 복사는 Clipboard API를 사용하며, 브라우저 지원 여부와 무관하게 폴백 UI를 제공할 필요는 없다(MVP 대상 환경 기준 지원됨).

## 수용 기준

- **SC-101** (`FR-101`): 고객 상세 화면에서 "새 프로젝트" 버튼 클릭 시 프로젝트 생성 폼에서 해당 고객이 미리 선택된 상태로 열린다.
- **SC-102** (`FR-102`): 의뢰 → 프로젝트 전환 대화상자에서 마감일을 입력하고 프로젝트를 생성하면 해당 마감일이 적용된 프로젝트가 생성된다.
- **SC-103** (`FR-103`): 프로젝트 목록에서 고객명으로 검색하면 해당 고객의 프로젝트만 표시된다. 프로젝트 제목으로 검색하면 해당 제목이 포함된 프로젝트만 표시된다. 검색어와 상태 필터를 동시에 적용할 수 있다.
- **SC-104** (`FR-104`): 프로젝트 상세 헤더에 전체 단계가 순서대로 시각적으로 표시되며, 현재 단계가 강조된다. 단계를 클릭하면 해당 단계로 변경되고 타임라인에 기록된다.
- **SC-105** (`FR-105`): 타임라인 탭에서 메모를 입력하고 저장하면 타임라인에 `MEMO` 타입의 이벤트로 추가된다.
- **SC-106** (`FR-106`, `FR-107`): 고객 포털에서 파일을 첨부하여 수정 요청 제출 시, 사업자 화면의 수정 요청 목록에서 해당 요청의 본문 아래에 "첨부 파일" 링크가 별도 영역으로 표시된다.
- **SC-107** (`FR-108`): 에셋 목록에서 삭제 버튼을 클릭하면 에셋이 삭제되고 목록에서 사라진다. 타임라인에 삭제 이벤트가 기록된다.
- **SC-108** (`FR-109`): 수정 요청 목록에서 삭제 버튼을 클릭하면 해당 수정 요청이 삭제되고 목록에서 사라진다.
- **SC-109** (`FR-110`): 공유 링크가 활성화된 프로젝트 상세에서 "링크 복사" 버튼 클릭 시 클립보드에 URL이 복사된다. 버튼 텍스트가 "복사됨"으로 일시 변경된다.
- **SC-110** (`FR-111`): 대시보드 상단에 현재 워크스페이스의 진행 중 프로젝트 수와 마감 임박(2일 이내) 프로젝트 수가 표시된다.

## 범위 외 (Out of Scope)

- 에셋·수정 요청 삭제 시 확인 대화상자 (MVP 단계에서는 즉시 삭제)
- 프로젝트 완료 처리 명시 UI (기존 `isProjectDone` 로직 유지)
- 수정 요청 복수 파일 첨부 (1개 파일만 지원하는 현재 구현 유지)
- 담당자별 프로젝트 필터
- 전역 검색 (고객·수정 요청 포함 통합 검색)
- 이메일·인앱 알림
- 메시지 템플릿 미리보기

---

## 미결 사항

없음. 현재 코드베이스 분석과 UX 검토를 통해 모든 항목이 결정되었다.
