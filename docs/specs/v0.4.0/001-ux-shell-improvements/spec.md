## 배경 및 목적

현재 Flowrit 대시보드는 기능적으로 충분하지만, 글로벌 셸(사이드바·헤더)과 핵심 페이지의 사용 흐름에 다음 공백이 있다.

1. **사용자 신원 미표시**: 사이드바에 누가 로그인되어 있는지 보이지 않고, 로그아웃 버튼이 UI 어디에도 없다.
2. **워크스페이스 컨텍스트 부재**: 헤더에 현재 워크스페이스 이름이 표시되지 않아 다중 워크스페이스 사용 시 혼란이 생길 수 있다. 사이드바의 "나의 작업실"은 실제 워크스페이스 이름이 아닌 하드코딩된 문자열이다.
3. **대시보드 KPI 부족**: 현황 요약 카드가 "진행 중"·"마감 임박" 2개뿐이어서, 즉시 처리가 필요한 수정 요청과 신규 접수 건수를 한눈에 파악하기 어렵다.
4. **빈 상태(Empty State) CTA 부재**: 프로젝트·고객·수정 요청 목록이 비어 있을 때, 다음 행동을 취할 수 있는 인라인 버튼이 없어 헤더까지 스크롤해야 한다.

이 스펙은 위 공백을 해소하여, 처음 접속한 팀원도 방향을 잃지 않고 작업을 시작할 수 있게 만드는 것을 목표로 한다.

## 사용자 스토리 (User Stories)

- **US-001**: 팀원으로서, 사이드바에서 내 이름과 로그아웃 버튼을 바로 볼 수 있어, 공용 PC에서도 쉽게 계정을 전환할 수 있다.
- **US-002**: 팀 관리자로서, 헤더에 현재 워크스페이스 이름이 표시되어, 어떤 워크스페이스에 접속해 있는지 즉시 확인할 수 있다.
- **US-003**: 작업자로서, 대시보드에서 미완료 수정 요청 건수와 신규 접수 건수를 KPI 카드로 확인하여, 오늘 처리할 일의 전체 규모를 한눈에 파악할 수 있다.
- **US-004**: 신규 사용자로서, 프로젝트·고객·수정 요청 목록이 비어 있을 때, 빈 상태 화면에서 바로 등록 버튼을 눌러 첫 항목을 만들 수 있다.

## 기능 요구사항 (Functional Requirements)

- **FR-001**: 사이드바 하단에 현재 로그인한 사용자의 이름과 이니셜 아바타를 표시한다.
- **FR-002**: 사이드바 하단에 로그아웃 버튼을 표시하며, 클릭 시 세션을 종료하고 로그인 페이지로 이동한다.
- **FR-003**: 대시보드 레이아웃 헤더(상단 고정 바)에 현재 워크스페이스 이름을 표시한다.
- **FR-004**: 대시보드 KPI 현황 요약 카드를 4개로 확장한다: 진행 중, 마감 임박(2일), 미완료 수정 요청, 신규 접수(PENDING 상태).
- **FR-005**: 프로젝트 목록 빈 상태에 "새 프로젝트" 버튼을 인라인으로 추가한다.
- **FR-006**: 고객 목록 빈 상태에 "고객 등록" 버튼을 인라인으로 추가한다.
- **FR-007**: 수정 요청 목록 빈 상태 안내 문구를 개선한다(프로젝트 목록 링크 제공).

## 비기능 요구사항 (Non-Functional Requirements)

- **NFR-001**: 사이드바에 사용자 정보를 추가해도 기존 레이아웃 240px 너비를 유지한다.
- **NFR-002**: 워크스페이스 이름 조회는 대시보드 레이아웃의 기존 `auth()` 호출과 함께 처리하여 추가 요청 왕복을 최소화한다.
- **NFR-003**: 로그아웃은 Server Action으로 구현하며, 자바스크립트 없이도 동작해야 한다.
- **NFR-004**: DESIGN.md의 디자인 원칙(compact, calm, 8px/12px 모서리, subtle hover)을 준수한다.

## 수용 기준 (Acceptance Criteria)

- **SC-001** (`FR-001` 관련): 사이드바 하단에 로그인 사용자의 이름과 이니셜 2자리 아바타(indigo soft 배경)가 표시된다.
- **SC-002** (`FR-002` 관련): 사이드바에 "로그아웃" 버튼이 있고, 클릭 시 세션이 종료되어 `/login`으로 리다이렉트된다.
- **SC-003** (`FR-003` 관련): 헤더 상단 고정 바 왼쪽 영역에 워크스페이스 이름이 표시된다.
- **SC-004** (`FR-004` 관련): 대시보드 현황 카드가 4개(`2×2` 그리드)로 표시되며, 각각 진행 중·마감 임박·미완료 수정 요청·신규 접수 건수를 보여준다.
- **SC-005** (`FR-005` 관련): 프로젝트 목록이 비어 있을 때, 빈 상태 패널 안에 `/projects/new`로 이동하는 "새 프로젝트" 버튼이 표시된다.
- **SC-006** (`FR-006` 관련): 고객 목록이 비어 있을 때, 빈 상태 패널 안에 `/customers/new`로 이동하는 "고객 등록" 버튼이 표시된다.
- **SC-007** (`FR-007` 관련): 수정 요청 목록이 비어 있을 때, "프로젝트에서 수정 요청을 등록하세요." 안내와 함께 `/projects` 링크가 표시된다.

## 범위 외 (Out of Scope)

- 워크스페이스 전환(multi-workspace switcher): 현재 단일 워크스페이스만 활성화되는 구조이므로 이번 스펙에서 제외한다.
- 사용자 프로필 드롭다운(이메일 표시, 프로필 편집 등): 사이드바 하단에 이름+로그아웃 버튼으로 충분하며, 드롭다운 복잡도는 이번 범위를 초과한다.
- 다크 모드: DESIGN.md에 정의되지 않은 테마.
- 모바일 반응형 사이드바(햄버거 메뉴): 별도 스펙으로 분리한다.
