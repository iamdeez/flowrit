## 배경 및 목적

기존 Flowrit에는 일반 문의 접수(`/intake/[slug]`)만 존재했다. 프리랜서 작업 특성상 단순 문의와 실제 작업 의뢰(주문서)는 성격이 다르다 — 주문서는 희망 날짜, 예산, 상세 의뢰 내용 같은 구조화된 정보를 필요로 한다.

또한 대시보드의 "신규 접수" 메뉴가 문의와 주문서를 구분 없이 섞어 표시하고 있어, 접수 건 관리가 어려웠다.

이 스펙은 세 가지를 해결한다:
1. 고객 전용 주문서 폼 페이지 (`/order/[workspaceSlug]`)
2. 문의·주문서를 통합 관리하는 "주문서 관리" 대시보드 (`/orders`)
3. Inquiry 모델에 `formType` 필드로 일반 문의와 주문서 구분

---

## 사용자 스토리 (User Stories)

- **US-001**: 프리랜서로서, 고객에게 주문서 링크를 따로 공유하여 구조화된 작업 의뢰를 받고 싶다.
- **US-002**: 프리랜서로서, 대시보드에서 일반 문의와 주문서를 탭으로 구분해 확인하고 싶다.
- **US-003**: 프리랜서로서, 접수된 의뢰를 삭제(무시)하거나 프로젝트로 전환할 수 있어야 한다.
- **US-004**: 고객으로서, 주문서 폼에서 이름·연락처·희망 날짜·예산·의뢰 내용·파일을 첨부해 제출하고 싶다.
- **US-005**: 프리랜서로서, 설정 페이지에서 일반 문의 폼과 주문서 폼 링크를 각각 복사할 수 있어야 한다.

---

## 기능 요구사항 (Functional Requirements)

- **FR-001**: 고객 전용 주문서 폼 페이지(`/order/[workspaceSlug]`)가 존재하며, 비인증 고객이 접근 가능해야 한다.
- **FR-002**: 주문서 폼 필드: 이름(필수), 연락처, 희망 날짜(date), 예산(text), 의뢰 내용(필수), 파일 첨부(최대 100개).
- **FR-003**: 주문서 제출 시 `Inquiry.formType = 'ORDER'`로 저장하고, 희망 날짜·예산은 content 상단에 구조화 prefix(`희망 날짜: ...\n예산: ...\n---\n`)로 기록한다.
- **FR-004**: 주문서 제출 후 워크스페이스 멤버 전체에게 `NEW_INQUIRY` 알림이 발송되어야 한다.
- **FR-005**: 대시보드 "주문서 관리" 페이지(`/orders`)가 존재하며, 탭(전체/일반 문의/주문서)과 상태 필터(대기 중/전환 완료)를 제공해야 한다.
- **FR-006**: 접수 건마다 "무시" 버튼으로 `status: DISMISSED` 처리할 수 있어야 한다.
- **FR-007**: 접수 건마다 기존 "프로젝트로 전환" 다이얼로그를 통해 Project로 전환할 수 있어야 한다.
- **FR-008**: "주문서 관리" 페이지 상단에 일반 문의 폼과 주문서 폼 링크를 표시해야 한다.
- **FR-009**: 사이드바에 "주문서 관리" 메뉴가 추가되고, 대기 중인 Inquiry 수 뱃지를 표시해야 한다.
- **FR-010**: 설정 페이지의 공유 링크 섹션에 일반 문의 폼(`/intake/[slug]`)과 주문서 폼(`/order/[slug]`) 링크를 각각 표시해야 한다.

---

## 비기능 요구사항 (Non-Functional Requirements)

- **NFR-001**: 주문서 폼 파일 업로드는 기존 presigned URL 방식을 따른다 (최대 100개, 개별 10MB).
- **NFR-002**: `formType` 필드는 DB 마이그레이션으로 기존 Inquiry 레코드에 `DEFAULT 'INQUIRY'`가 적용되어야 한다.
- **NFR-003**: Inquiry `status`의 `DISMISSED` 값은 새로운 필터 UI 없이 `PENDING`과 구분된 별도 상태로 저장되어야 한다 (대시보드 필터에는 DISMISSED 탭 미노출).

---

## 수용 기준 (Acceptance Criteria)

- **SC-001** (`FR-001`): `/order/{slug}` 접근 시 비인증 상태에서 주문서 폼 표시.
- **SC-002** (`FR-002`, `FR-003`): 주문서 폼 제출 시 `Inquiry.formType = 'ORDER'`, content에 희망 날짜·예산 prefix 포함.
- **SC-003** (`FR-004`): 주문서 제출 시 워크스페이스 멤버에게 `NEW_INQUIRY` 알림 발송.
- **SC-004** (`FR-005`): `/orders` 페이지에서 탭(전체/일반 문의/주문서) 전환 및 상태 필터 동작.
- **SC-005** (`FR-006`): "무시" 버튼 클릭 후 confirm 시 `status: DISMISSED`로 업데이트되고 목록에서 제거.
- **SC-006** (`FR-007`): "프로젝트로 전환" 다이얼로그가 orders 페이지에서도 정상 동작.
- **SC-007** (`FR-009`): 사이드바 "주문서 관리" 메뉴에 PENDING Inquiry 수 뱃지 표시.
- **SC-008** (`FR-010`): 설정 페이지 공유 링크 섹션에 두 폼 링크 각각 복사 버튼 포함.

---

## 범위 외 (Out of Scope)

- DISMISSED 건의 별도 조회 페이지
- 주문서 폼의 사용자 정의 필드 추가 (항목 고정)
- 주문서 제출 후 고객에게 이메일 자동 발송
- 일반 문의 폼(`/intake`)의 필드 변경 — 기존 폼 유지
