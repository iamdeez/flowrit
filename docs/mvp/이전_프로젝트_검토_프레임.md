## 4. 고객 메시지 분석

실제 고객 메시지를 익명화해서 붙여넣는다. 고객의 표현은 랜딩페이지와 제품 문구의 원재료가 된다.

```text
예:
"지금 어디까지 됐나요?"
"지난번에 보낸 링크 다시 받을 수 있을까요?"
"3번 사진 얼굴을 조금 더 밝게 해주세요."
"최종본은 언제 받을 수 있나요?"
```

| 메시지 | 의미 | 제품 기능으로 바꾸면 |
|---|---|---|
|  | 진행상황 문의 | 고객용 진행상황 페이지 |
|  | 파일 링크 재요청 | 파일 링크 관리 |
|  | 수정 요청 | 수정 요청 DB |
|  | 납품 일정 확인 | 마감일/상태 표시 |

## 5. 시스템으로 바꿔야 할 업무

| 업무 | 현재 방식 | 시스템화 방향 | 우선순위 |
|---|---|---|---|
| 고객별 상태 확인 | 머릿속/메모/카톡 | 프로젝트 상태 보드 | 높음 |
| 수정 요청 정리 | 카톡 대화 확인 | 수정 요청 목록 | 높음 |
| 파일 링크 관리 | 드라이브 링크 수동 검색 | 프로젝트별 링크 저장 | 높음 |
| 고객 안내 | 카톡 직접 작성 | 메시지 템플릿 복사 | 중간 |
| 고객 진행상황 공유 | 직접 답변 | 공유 페이지 | 높음 |
| 납품 완료 처리 | 기억/메모 | 완료 상태 및 보관 | 중간 |

## 6. MVP 판단 기준

아래 조건 중 3개 이상에 해당하면 MVP 핵심 기능으로 반영한다.

- 같은 문제가 2회 이상 반복되었다.
- 문제 해결에 월 3만~5만원 이상 지불할 가능성이 있다.
- 수동으로 처리할 때 시간이 반복적으로 낭비되었다.
- 고객 불만 또는 불안으로 이어졌다.
- 납품 품질이나 일정에 영향을 줬다.
- 노션 템플릿으로 먼저 해결해볼 수 있다.

## 7. MVP에 넣을 것 / 뺄 것

### 넣을 것

| 기능 | 포함 이유 | 관련 문제 |
|---|---|---|
| 고객 등록 | 고객별 업무 흐름 관리의 기준 | 고객 정보 분산 |
| 프로젝트 등록 | 의뢰 단위 관리 | 진행상황 불명확 |
| 단계 상태 관리 | 현재 어디까지 왔는지 표시 | 진행상황 문의 |
| 수정 요청 등록 | 요청 누락 방지 | 수정 요청 묻힘 |
| 파일 링크 등록 | 납품 링크 재전송 감소 | 링크 분산 |
| 고객용 진행상황 페이지 | 고객 문의 감소 | 상태 문의 반복 |
| 메시지 복사 버튼 | 카카오톡 옆에서 사용 | 응대 반복 |

### 뺄 것

| 기능 | 제외 이유 | 재검토 시점 |
|---|---|---|
| 대용량 파일 저장 | 비용 리스크가 크고 핵심 검증과 무관 | 유료 고객 30곳 이후 |
| 카카오톡 자동 연동 | 초기 개발 난이도와 정책 리스크가 큼 | 수동 사용 반복 확인 후 |
| 고급 AI 챗봇 | 핵심 문제 검증 전에는 과함 | 수정 요청 데이터 축적 후 |
| 복잡한 자동화 빌더 | 초기 사용자가 설정하기 어려움 | 반복 자동화 패턴 확인 후 |
| 모든 업종 템플릿 | 메시지가 흐려짐 | 첫 업종 PMF 신호 이후 |

## 8. 유료 파일럿 제안으로 바꾸기

### 파일럿 상품명

고객 의뢰·수정요청·납품관리 세팅 패키지

### 제공 범위

- 고객/프로젝트 관리 보드 세팅
- 진행 단계 템플릿 세팅
- 수정 요청 관리표 세팅
- 납품 링크 관리표 세팅
- 고객 공유용 진행상황 페이지 세팅
- 카카오톡 안내 메시지 템플릿 제공

### 가격 가설

| 상품 | 가격 | 목적 |
|---|---:|---|
| 베타 세팅 | 100,000원~300,000원 | 실제 지불 의향 검증 |
| 월 사용/관리 | 29,000원~49,000원 | 반복 사용 검증 |
